# TS.3 Back-End (Web Server Services) - Complete Framework

## **Domain: Back-End (Web Server Services) (TS.3)**
**Description**: I can utilize Web Server Services when creating a back-end application

---

## **Competency 1: Consume Application Programming Interfaces (TS.3.1)**
**Description**: I can understand Application Programmable Interfaces and how they work

### **Skill 1.1: Understand API concepts (TS.3.1.1) - Level 1**
**Indicator**: Can explain what APIs are and how they function
**Assessment**: Student can describe REST principles, HTTP methods, and API endpoints

### **Skill 1.2: Make API requests (TS.3.1.2) - Level 2**
**Indicator**: Can send and receive data through API calls
**Assessment**: Student can use tools like Postman or code to make GET, POST, PUT, DELETE requests

### **Skill 1.3: Handle API responses (TS.3.1.3) - Level 2**
**Indicator**: Can process and utilize API response data
**Assessment**: Student can parse JSON responses, handle errors, and integrate data into applications

---

## **Competency 2: Handle web servers (TS.3.2)**
**Description**: I can handling request/response (web & form submissions) to create dynamic web applications

### **Skill 2.1: Set up web servers (TS.3.2.1) - Level 1**
**Indicator**: Can configure basic web server environments
**Assessment**: Student can install and configure web servers like Apache, Nginx, or Node.js

### **Skill 2.2: Process HTTP requests (TS.3.2.2) - Level 2**
**Indicator**: Can handle different types of web requests
**Assessment**: Student can create routes, process form data, and manage session information

### **Skill 2.3: Implement server-side logic (TS.3.2.3) - Level 3**
**Indicator**: Can build complex server-side applications
**Assessment**: Student can create middleware, implement authentication, and manage application state

---

## **Competency 3: Configure servers (TS.3.3)**
**Description**: I can configure Object Relational Mapping (ORM) and other back-end services

### **Skill 3.1: Set up databases (TS.3.3.1) - Level 1**
**Indicator**: Can install and configure database systems
**Assessment**: Student can set up MySQL, PostgreSQL, or MongoDB databases

### **Skill 3.2: Implement ORM systems (TS.3.3.2) - Level 2**
**Indicator**: Can use Object Relational Mapping tools
**Assessment**: Student can configure and use ORMs like Sequelize, Mongoose, or SQLAlchemy

### **Skill 3.3: Optimize server performance (TS.3.3.3) - Level 3**
**Indicator**: Can configure servers for optimal performance
**Assessment**: Student can implement caching, load balancing, and performance monitoring

---

## **Competency 4: Design systems & architecture (TS.3.4)**
**Description**: I can create logical system architecture or database design using industry standards

### **Skill 4.1: Create database schemas (TS.3.4.1) - Level 2**
**Indicator**: Can design normalized database structures
**Assessment**: Student can create ERDs, define relationships, and implement database schemas

### **Skill 4.2: Plan system architecture (TS.3.4.2) - Level 2**
**Indicator**: Can design scalable system architectures
**Assessment**: Student can create system diagrams, plan microservices, and design API structures

### **Skill 4.3: Implement design patterns (TS.3.4.3) - Level 3**
**Indicator**: Can apply industry-standard architectural patterns
**Assessment**: Student can implement MVC, repository patterns, and design for scalability

---

**Total TS.3 Structure:**
- 4 Competencies
- 12 Skills (3 Level 1, 6 Level 2, 3 Level 3)

# TS.4 Back-End (Databases) - Complete Framework

---

## **Competency 1: Create a data structure (TS.4.1)**
**Description**: I can use programming techniques to create a data structure for organizing information

### **Skill 1.1: Design data models (TS.4.1.1) - Level 1**
**Indicator**: Can identify entities and their attributes
**Assessment**: Student can analyze requirements and create basic data models

### **Skill 1.2: Define relationships (TS.4.1.2) - Level 2**
**Indicator**: Can establish connections between data entities
**Assessment**: Student can implement one-to-one, one-to-many, and many-to-many relationships

### **Skill 1.3: Normalize data structures (TS.4.1.3) - Level 3**
**Indicator**: Can optimize data structures for efficiency
**Assessment**: Student can apply normalization rules and eliminate data redundancy

---

## **Competency 2: Navigate data structures (TS.4.2)**

### **Skill 2.1: Write basic queries (TS.4.2.1) - Level 1**
**Indicator**: Can create simple SELECT, INSERT, UPDATE, DELETE statements
**Assessment**: Student can write basic SQL queries to retrieve and modify data

### **Skill 2.2: Implement complex queries (TS.4.2.2) - Level 2**
**Indicator**: Can use joins, subqueries, and aggregate functions
**Assessment**: Student can write advanced SQL with multiple tables and complex conditions

### **Skill 2.3: Optimize query performance (TS.4.2.3) - Level 3**
**Indicator**: Can improve database query efficiency
**Assessment**: Student can use indexes, analyze query plans, and optimize database performance

---

## **Competency 3: Understand data models (TS.4.3)**

### **Skill 3.1: Model business requirements (TS.4.3.1) - Level 2**
**Indicator**: Can translate business needs into data structures
**Assessment**: Student can analyze business processes and create appropriate data models

### **Skill 3.2: Implement referential integrity (TS.4.3.2) - Level 2**
**Indicator**: Can ensure data consistency and validity
**Assessment**: Student can use foreign keys, constraints, and validation rules

### **Skill 3.3: Design for scalability (TS.4.3.3) - Level 3**
**Indicator**: Can create data models that support growth
**Assessment**: Student can design partitioning strategies, handle large datasets, and plan for scaling
